# Web 母音化の最初の設計

## 概要

この文書は、Web プロトタイプを単純な連続音から、母音らしい音へ寄せるための最初の設計メモである。

初期段階では、声道の詳細な幾何形状モデルや高忠実度な物理モデルには進まない。まずは、現在の最小音声生成系を拡張して、励振源と簡易的なフォルマント近似フィルタを組み合わせる構成を採用する。

## この段階の目的

この段階で目指すのは、次のような状態である。

1. 連続音が、単なる発振器音よりも母音らしく聞こえる
2. 母音の違いを、少数のパラメータで切り替えられる
3. 実装がシンプルで、後から差し替えや拡張がしやすい

## 採用する構成

最初の構成は、以下のようにする。

1. 励振源として周期的な波形を使う
2. その出力を複数のバンドパス系フィルタへ並列に通す
3. 各フィルタを、母音のフォルマント近似として扱う
4. 出力を合成して音声出力へ送る

これは厳密な発声器官シミュレーションではないが、母音らしさを最小コストで導入する入口としては妥当である。

## 最初に扱うパラメータ

初期段階では、パラメータを増やしすぎない。

- `frequency`: 基本周波数
- `gain`: 出力音量
- `vowel`: 母音プリセット
- `tractScale`: フォルマント全体のスケール

この段階では、`tractScale` を用いて、声道長の変化を粗く近似する。厳密な意味での声道長モデルではないが、母音の印象変化を観察するには十分である。

## 最初に扱う母音プリセット

まずは少数の代表的な母音に絞る。

- `a`
- `i`
- `u`
- `e`
- `o`

それぞれに対して、F1, F2, F3 の近似値を持たせる。

## 実装方針

初期実装では、次のように進める。

1. 現在の単一オシレータ出力を残しつつ、フィルタバンク構成へ置き換える
2. 母音プリセット定義を、音声エンジン本体から分離する
3. UI に母音選択と声道スケール操作を加える
4. まずは「違いが分かる」ことを優先し、忠実度は次段階で詰める

## この段階でまだやらないこと

以下は次の段階に回す。

- 声道断面積列の直接制御
- Pink Trombone 風の形状操作
- 子音や乱流の導入
- 動的な共調音
- 高忠実度な声帯モデル

## この設計の利点

- 現在の最小構成を大きく崩さずに進められる
- 母音らしさを早く導入できる
- UI と音響モデルの接続が単純
- 後でより物理的なモデルへ差し替える際の足場になる

## 備考

この文書は、Web 側の初期母音化に向けた第一段階の設計メモであり、今後の試聴結果や研究環境での比較に応じて更新する前提の草稿である。
