## 概要

`web/prototypes/waveguide-vocal-tract/` に第3プロトタイプを実装する。
第2プロトタイプ（`tube-vocal-tract`）で確立した UI・構造を引き継ぎつつ、音響シミュレーションを正確に実装し直す。具体的には①セクション遅延の正確な実装、②日本語母音の実測断面積データ、③LF モデル声帯音源、④損失モデルの4点を改修する。

## 改修1：セクション遅延とダブルバッファ

### 問題

第2プロトタイプでは `stepTube()` 内のループが左から右へ順番に波を更新するため、グロッタルの入力が同じサンプル内に口端まで伝わっていた。これは遅延なしの「瞬時伝搬」であり、管の共鳴周波数が正しく再現されない。

### 解法：ダブルバッファリング

各サンプル更新時に現在の波の状態をコピーしてから散乱計算を行う。これにより1セクションを伝搬するのに1サンプル分の時間がかかる。

```
// 更新前の状態を保存
const oldRight = right.slice();
const oldLeft  = left.slice();

// 旧状態を使って散乱を計算
for (let i = 0; i < N-1; i++) {
  right[i+1] = (1+k[i]) * oldRight[i] - k[i] * oldLeft[i+1];
  left[i]    = (1-k[i]) * oldLeft[i+1] + k[i] * oldRight[i];
}
// 境界条件
right[0]   = glottalInput + glottalReflect * oldLeft[0];
left[N-1]  = lipReflect * oldRight[N-1];
// 出力: 口端の全圧力
output = oldRight[N-1] + oldLeft[N-1];
```

グロッタルの入力が口端に到達するまでに N サンプルかかるようになり、声道の共鳴が物理的に正しく再現される。

### セクション数の決定

1セクションあたり1サンプルになるようセクション数を決める。

```
N = round(L × fs / c)
  = round(17 cm × 44100 Hz / 35000 cm/s)
  = round(21.44)
  = 21 セクション
```

サンプルレートに応じて動的に算出する（44100 Hz → 21、48000 Hz → 23）。

## 改修2：日本語母音の実測断面積データ

### データソース

Maeda (1979) による日本語母音の X 線トレース計測値、および Story (2005) の MRI 計測断面積の特性を参照して算出した。各母音の物理的特徴を以下に示す。

| 母音 | 声道の形状的特徴 | 目標フォルマント |
|------|----------------|---------------|
| /a/ | 咽頭広・口腔広・口元やや狭 | F1≈730, F2≈1090, F3≈2440 Hz |
| /i/ | 口腔前部が極めて狭い（前舌高位）、咽頭広 | F1≈270, F2≈2290, F3≈3010 Hz |
| /u/ | 口腔後部が狭い（後舌高位）、唇丸め | F1≈300, F2≈870, F3≈2240 Hz |
| /e/ | /i/ に近いが開口度大、前部やや広い | F1≈530, F2≈1840, F3≈2480 Hz |
| /o/ | 後部収縮＋唇丸め、咽頭は /a/ より狭い | F1≈570, F2≈840, F3≈2410 Hz |

断面積は cm²、グロッタル側（咽頭後壁）から口側（口唇）に向けて 22 値で定義する。第2プロトタイプの 16 値から増加する。

### フォルマント検証

実装後、`tube-model.ts` の伝達関数ピーク推定（改良版、後述）で F1〜F3 を確認し、上記目標値との誤差が大きい母音は断面積を手調整する。

## 改修3：LF モデル励振源

### のこぎり波との違い

のこぎり波はすべての倍音を等振幅で持つ。実際の声帯は開口位相・閉鎖位相の非対称性があり、高周波倍音の減衰プロファイルが異なる。LF モデルはその形状を数式で記述する。

### LF モデルの式

開口相（0 ≤ t ≤ Te）の声門流速微分：

```
u'(t) = E0 · exp(α·t) · sin(ωg · t)
ωg = π / Tp
```

閉鎖後の戻り相（Te < t ≤ T0）：

```
u'(t) = -Ee / (ε · Ta) · (exp(-ε·(t - Te)) - exp(-ε·(T0 - Te)))
```

パラメータ（標準的な平常声）：

| パラメータ | 意味 | 標準値 |
|-----------|------|-------|
| Rg | 開口比（= Tp / (T0/2)） | 1.0 |
| Rk | 非対称比（= (Te - Tp) / Tp） | 0.3 |
| Ra | 急速閉鎖の緩和時定数比（= Ta / T0） | 0.01 |

これらは無次元パラメータで、ピッチ周期 T0 から各絶対時間を算出する。 `α` は周期内でのエネルギー保存条件（積分ゼロ）から数値的に解く。

### AudioWorklet 内での実装

各サンプルで位相カウンタ（0〜1）を進め、区間に応じて上記の式から音源サンプルを計算する。α の計算はパラメータ変化時（ピッチ変更時）のみ数値解法（ニュートン法、数十回の収束）で更新する。

## 改修4：損失モデル

### 損失の種類

声道内での損失には大きく2種類ある：
1. **粘性・熱損失**：管壁との境界層での摩擦と熱伝導
2. **壁振動損失**：軟組織の振動による吸収（低周波で顕著）

### 実装方針（定数近似）

厳密な周波数依存損失（IIR フィルター）は実装コストが高い。本プロトタイプでは以下の簡易近似を採用する。

- 各セクション通過ごとに波の振幅に損失係数を掛ける

```
lossPerSection = 1.0 - alpha
```

`alpha` は断面積に依存し、狭い管ほど損失が大きい（表面積/体積比が高い）：

```
alpha(A) = alphaBase / sqrt(A)   // A は断面積 cm²
alphaBase ≈ 0.001
```

壁振動損失は低周波成分に影響するため、別途 1次 IIR ローパスフィルターで近似する（カットオフ 300 Hz 程度のわずかな減衰）。

## フォルマント推定の改良

第2プロトタイプの 1/4 波長近似に代わり、伝達関数の極を求める方式に変更する。

### 実装：インパルス応答 → LPC → 極

1. 反射係数から声道フィルターにインパルスを入力し、短い応答（例: 512 サンプル）を記録
2. その応答に対して LPC (線形予測) 分析を適用（次数 2×(F1〜F4 分の数) = 8〜10 程度）
3. LPC 係数の根（複素極）を求め、正の周波数成分をフォルマントとして返す

LPC はメインスレッドで実行（AudioWorklet 外）し、断面積が変化するたびに再計算する。

## モジュール構成

第2プロトタイプから変更されるファイルのみ示す。UI ファイル（`tract-editor.ts`、`spectrum-view.ts`、`styles.css`）は流用する。

```
waveguide-vocal-tract/
├── package.json
├── tsconfig.json
├── vite.config.ts
├── index.html
├── public/
│   └── worklet/
│       └── waveguide-processor.js   ← 全面書き直し（遅延＋損失＋LF音源）
└── src/
    ├── main.ts                      ← 変更なし
    ├── styles.css                   ← 流用
    ├── audio/
    │   ├── engine.ts                ← セクション数変更・LFパラメータ追加
    │   ├── tube-model.ts            ← ダブルバッファ化・損失計算追加
    │   ├── lf-source.ts             ← 新規：LF モデルパラメータ計算
    │   ├── formant-estimator.ts     ← 新規：LPC によるフォルマント推定
    │   └── vowel-presets.ts         ← 実測値ベースに更新・22セクション化
    └── ui/
        ├── app.ts                   ← LF パラメータ UI を追加
        ├── tract-editor.ts          ← 流用（セクション数変更のみ）
        └── spectrum-view.ts         ← 流用
```

### 主要クラス・責務

#### `lf-source.ts` — `LFSource`
- `Rg`、`Rk`、`Ra` パラメータから LF 波形の各定数を算出
- α の計算はニュートン法で収束（反復 50 回程度）
- `nextSample(phase: number): number` で位相から波形値を返す

#### `formant-estimator.ts` — `estimateFormants()`
- 反射係数配列を受け取り、ステップ応答列を生成
- Levinson–Durbin アルゴリズムで LPC 係数を算出
- 根の計算で複素極を求め、フォルマント周波数を返す

#### `waveguide-processor.js`（AudioWorklet）
- ダブルバッファリングによる波動伝搬
- 各セクションで損失係数を適用
- LF モデルで声源信号を生成
