# 初期フェーズ計画

## 概要

この文書は、`voice-simulator` の初期フェーズで何を作るかを定義するための計画メモである。現時点では、最終目標である高度な発声器官シミュレーターを直接作り始めるのではなく、リアルタイム性、観察可能性、拡張可能性を備えた基礎システムを構築することを優先する。

初期フェーズの中心方針は、軽量な声帯モデルと声道モデルを用いたインタラクティブな音声生成系をまず成立させ、そのうえで解析、可視化、制御の基盤を整えることである。

## 初期フェーズの目的

初期フェーズでは、以下を達成することを目的とする。

1. 発声器官シミュレーションの最小構成をソフトウェアとして成立させる
2. モデルの挙動を見て、聞いて、調整できる状態を作る
3. 将来の高忠実度化や 3D 化に耐えられる構造を用意する
4. 文献調査や比較研究の結果を実装に反映しやすい形にする

## 初期フェーズで作るもの

初期フェーズでは、次の要素を優先的に作る。

### 1. 最小の音声生成コア

最初に必要なのは、音を出せる核である。ここでは、以下を含む軽量な音声生成コアを作る。

- 声帯に相当する簡略化された励振モデル
- 声道に相当する軽量な伝搬・共鳴モデル
- 母音を中心とした基本的な音声生成

この段階では、完全な生体再現や高忠実度よりも、まずシンプルなモデルで安定して音が出て、挙動が理解しやすいことを優先する。

### 2. インタラクティブな制御系

モデルを研究・開発するうえで、パラメータ変更に対する音の変化を即座に確認できることは重要である。そのため、初期段階から以下のような制御系を持たせる。

- 声帯パラメータの調整
- 声道形状またはその近似パラメータの調整
- 発音状態の切り替え
- リアルタイム試行のための UI または操作インターフェース

この制御系は、最終的な製品 UI ではなく、研究・実験用の操作環境として考える。

### 3. 可視化

本プロジェクトでは、音を生成するだけでなく、構造や状態を理解できることが重要である。そのため、初期フェーズから可視化を組み込む。

- 声道形状またはその近似表現の可視化
- パラメータ変化の可視化
- 波形、スペクトル、フォルマントなどの観察

可視化は、モデルの妥当性を議論するための共通の観察面としても役立つ。

### 4. 解析ツール

初期フェーズでは、生成音を評価するための簡易な解析機能を持たせる。

- スペクトル表示
- フォルマント観察
- 基本周波数や声質指標の観察
- 比較用の音声データを読み込んで参照できる仕組み

解析手法は、モデルの正しさを断定するためではなく、調整、比較、観察、デバッグのための道具として扱う。

### 5. 実験しやすいモジュール構成

初期段階から、機能をできるだけ分離しておく。

- 励振源
- 声道伝搬・共鳴
- 乱流や子音生成の候補領域
- 解析
- 可視化
- UI / 操作系

こうしておくことで、後からモデル差し替えや高忠実度化をしやすくする。

## 初期フェーズでは後回しにするもの

以下は重要ではあるが、初期フェーズの主目標にはしない。

### 1. 高忠実度 3D シミュレーション

3D 化や高精度な波動シミュレーションは将来の重要テーマだが、初期段階では重すぎる。まずは 1D/2D またはそれに準ずる軽量モデルを優先する。

### 2. 完全な子音・乱流・鼻腔結合の再現

子音、摩擦音、乱流、鼻腔との結合は難所であり、ここを最初から包括的に扱うと進行が遅くなる。初期フェーズでは、母音と単純な発音変化を中心に据える。

### 3. 素材や軟組織特性の高精度モデル

声道壁の柔らかさや材質差などは、本プロジェクトの将来的な差別化要素になりうるが、初期段階では概念整理と調査を優先し、実装は後続フェーズで扱う。

### 4. 残響や空間音響の本格統合

残響や空間伝搬は関連が深いが、発声器官シミュレーション本体とは責務を分けるべきである。必要なら後段の音響モジュールとして追加する。

## 初期フェーズの具体的な到達点

初期フェーズの成果として、少なくとも以下の状態を目指す。

1. 基本的な母音に相当する音を生成できる
2. 声帯や声道の主要パラメータを変更し、その結果を音として確認できる
3. 生成音の波形やスペクトルを確認できる
4. 生成コア、可視化、解析、UI が分離された構造になっている
5. 今後どこに高忠実度化や新機能を差し込むかが見える

## 初期フェーズの設計原則

初期フェーズでは、以下の原則で進める。

### 軽量であること

リアルタイムに動かせること、繰り返し試せることを重視する。重い高精度モデルは検証や将来拡張の対象として扱う。

### 観察しやすいこと

音が出るだけでは不十分であり、何が起きているかが見えることを重視する。

### 置き換えやすいこと

一度作った簡易モデルを後で差し替えられるように、モジュール境界を明確にする。

### 比較しやすいこと

参考音声、解析結果、文献上の知見と照らし合わせやすい構造を意識する。

## 現時点で有力な進め方

現時点では、次の順序が有力に見える。

1. 最小の励振源と声道モデルを実装して、まず音を出す
2. そのモデルを操作するための簡易 UI またはパラメータ操作環境を作る
3. 波形、スペクトル、フォルマントなどの観察機能を加える
4. 母音の安定化を進める
5. その後、子音、乱流、鼻腔、材質、3D 検証などの拡張候補に進む

## この段階で保留している論点

以下は初期フェーズに入る前後で決める必要があるが、まだ確定していない。

- 実装言語と主要ライブラリ
- 音声処理系と可視化系を同一環境で行うかどうか
- ブラウザ中心で進めるか、ネイティブ寄りで進めるか
- 2D 表現をどの程度重視するか
- 将来の 3D 拡張を見据えたデータ構造をどうするか

## 備考

この文書は、プロジェクト初期段階の方針を固めるための草稿であり、実装や追加調査の結果に応じて更新される前提の計画文書である。
