# 調査・実装メモ：2D 線形音響 FDTD プロトタイプ

作成日：2026-04-24

## 目的

次のプロトタイプとして、2D グリッド上の音響伝搬を可視化し、耳セルで観測した音圧を再生できる Web アプリを作る。

この段階では声道形状の高忠実度再現ではなく、以下を優先する。

- 音圧 `p` と粒子速度 `vx`, `vy` を分けて持つ
- staggered grid FDTD に近い時間発展を行う
- 障害物、吸音、透過、散乱、周波数依存壁をセルとして配置できる
- 外周の単純吸音層で端反射を抑える
- ソースセルと耳セルを自由に置き、耳セルの圧力波形を録音・再生する

## 参照した既存資産

ローカルでは、`docs/plans/digital-waveguide-mesh-prototype-plan.md` と `research/scripts/dwm_solver.py` を確認した。既存 DWM は散乱接続で 2D 音場を扱う研究側プロトタイプであり、今回の FDTD は Web 側で視覚的・対話的に扱う別系統のプロトタイプとして置くのが自然である。

既存方針と合わせ、今回も最初から高精度な声道音響モデルに寄せすぎず、数値安定性、観察しやすさ、後続実験への拡張余地を優先する。

- ScienceDirect 掲載の staggered-grid acoustic FD 論文スニペットでは、2D の一次音響方程式を `P`, `vx`, `vz` で離散化する一般形が示されている。  
  https://www.sciencedirect.com/science/article/pii/S0021999118305333

## 採用する方程式

連続体の線形音響方程式を、密度 `rho` と音速 `c` が一定の空気として扱う。

```text
∂vx/∂t = -(1/rho) ∂p/∂x
∂vy/∂t = -(1/rho) ∂p/∂y
∂p/∂t  = -rho c^2 (∂vx/∂x + ∂vy/∂y)
```

Web 実装では、表示と操作を軽くするため、物理単位の厳密性より安定性を優先し、正規化係数として実装する。

## グリッド配置

staggered grid は次の配置にする。

```text
p[i,j]      : セル中心、サイズ width * height
vx[i,j]     : 縦方向セル面、サイズ (width + 1) * height
vy[i,j]     : 横方向セル面、サイズ width * (height + 1)
material[i,j]: セル種別
```

速度を先に更新し、その後に速度の発散から圧力を更新する。

## 安定条件

2D の陽解法では、概ね Courant 条件

```text
c dt / dx <= 1 / sqrt(2)
```

を満たす必要がある。今回の Web 実装では `courant = 0.45` 程度に固定し、見た目の滑らかさのために `requestAnimationFrame` ごとに複数サブステップを進める。

## セル種別

最初の実装では、セルは以下の材料に分ける。

| 種別 | 表示 | 実装方針 |
|---|---|---|
| 空気 | 白〜赤/青 | 通常の更新対象 |
| 完全反射壁 | 黒 | 壁セル内部を非更新にし、壁をまたぐ法線速度を 0 にする |
| 吸音壁 | 暗い灰 | 更新対象だが圧力・速度を強めに減衰する |
| 透過壁 | 半透明灰 | 更新対象だがやや減衰し、波を弱く通す |
| 散乱壁 | 斜線調の灰 | 粗い反射・透過を交互に混ぜ、方向性を乱す近似 |
| 周波数依存壁 | 紫 | 圧力の一次ローパス状態を持ち、高周波成分だけ強く減衰する |
| 音源 | 緑 | 指定位置に正弦波バーストまたはインパルスを加算 |
| 耳 | 黄 | 指定位置の圧力を記録 |

厳密な壁インピーダンス境界や完全な散乱境界は今後の課題とし、最初は操作して違いが見えるモデルを優先する。

## 外周吸音層

グリッド端には単純なスポンジ層を置く。端からの距離に応じて減衰係数を強くし、`p`, `vx`, `vy` に毎ステップ掛ける。

PML ではないため斜入射の反射は残るが、初期プロトタイプとしては十分である。将来的には split-field PML または CPML を検討する。

## 録音と再生

耳セルの `p` をシミュレーション内部ステップごとに記録し、Web Audio API の `AudioBuffer` に変換して再生する。可聴化のため、録音バッファは最大振幅で正規化し、再生時にゲインを控えめにする。

注意点：

- 画面用のグリッド解像度と音声用サンプルレートは一致しない
- 初期版では「物理的に正しい音」より、伝搬した波を音として確認できることを優先する
- 後続では固定サンプルレートの Worker / AudioWorklet 化を検討する

## 実装場所

新規 Web プロトタイプとして以下に配置する。

```text
web/prototypes/acoustic-fdtd-2d/
```

主なファイル：

- `src/sim/fdtd.ts`：FDTD 状態、更新、材料モデル
- `src/audio/recording.ts`：耳セル録音の再生
- `src/ui/app.ts`：Canvas 可視化、ツール、操作 UI
- `src/styles.css`：2D シミュレータ向け UI

## 初期成功条件

- Canvas 上で赤/青の圧力波が伝搬する
- 音源、耳、各種壁をマウスで配置できる
- 反射壁で反射が見える
- 吸音層と吸音壁で波が弱まる
- 耳セルで記録した信号を再生できる
- `npm run build` が通る

## 後続課題

- 壁セルをセル中心材料ではなく面境界条件として扱う
- 物理単位に基づく `dx`, `dt`, `rho`, `c` の整理
- PML / CPML の導入
- インピーダンス境界による周波数依存吸音
- Worker 化による UI と計算の分離
- 録音サンプルレートと FDTD 内部ステップのリサンプリング
- 声道形状プリセット、口唇放射、声門音源との接続
