# プロトタイプ開発方針

作成日：2026-04-11

## 第4プロトタイプ以降の基本方針

### リアルタイム性よりも精度と使いやすさを優先する

これまでのプロトタイプ（第1〜第3）はブラウザ上でのリアルタイム合成（AudioWorklet による低レイテンシ処理）を前提に設計されてきた。しかし今後は **リアルタイム性を必須条件としない**。

#### 目指す UX

```
パラメータ設定 → 「生成」ボタン → 波形バッファを計算（数秒かかっても可）→ 「再生」ボタンで聴く
```

WAV ファイルのダウンロードができればなお良い。

#### この方針変更で得られるメリット

| 制約の解放 | 得られる自由 |
|---|---|
| AudioWorklet のリアルタイム制約 | 計算コストの高いモデルを使える |
| ブラウザの 1 フレーム（≈1.4ms）以内に処理を終わらせる必要がない | Python や OfflineAudioContext でオフライン処理ができる |
| セクション数・オーバーサンプリング倍率の上限 | セクション数を増やせる、高精度 ODE ソルバーが使える |

#### 実装候補アプローチ

1. **JavaScript + `OfflineAudioContext`**
   既存の波動管コードをそのまま利用しつつ、リアルタイム制約を外して `Float32Array` に書き出し → `AudioBuffer` に変換して `<audio>` タグで再生。

2. **JavaScript + 純粋な配列計算**
   Web Audio API を一切使わずに `Float32Array` を直接計算し、Blob URL として再生・ダウンロード。

3. **Python (scipy / numpy)**
   研究ワークスペース（`research/`）側で波形を生成し WAV 出力。Jupyter Notebook から `IPython.display.Audio` で再生。

#### 精度向上のために検討すべき要素

- **非線形声帯モデル**（Rosenberg パルス、完全 LF モデル、2 質量モデルなど）
- **鼻腔分岐路**（鼻音・鼻母音のシミュレーション）
- **放射インピーダンスの精密化**（単純反射係数 −1 を超えた実装）
- **摩擦音生成**（乱流ノイズ注入）
- **セクション数の増加**（44 → 88 以上）
- **4 次 Runge-Kutta 法など高精度時間積分**
